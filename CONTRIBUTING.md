# Contribuyendo al Proyecto

Gracias por tu interés en contribuir a Football Tournament Tracker.

## 📋 Código de Conducta

Por favor, sé respetuoso y constructivo en todas las interacciones.

## 🐛 Reportar Bugs

Si encuentras un bug, por favor crea una issue con:
- Descripción clara del problema
- Pasos para reproducirlo
- Resultado esperado vs actual
- Screenshots o videos si es posible
- Versión de Flutter y plataforma

## ✨ Solicitar Características

Para solicitar nuevas características:
1. Verifica que no exista ya
2. Describe claramente la funcionalidad
3. Explica por qué es útil
4. Proporciona ejemplos de uso

## 🔧 Desarrollo

### Configuración del Entorno

1. Fork el repositorio
2. Clone tu fork: `git clone https://github.com/TU_USUARIO/football-tournament-tracker.git`
3. Crea una rama de feature: `git checkout -b feature/tu-feature`
4. Realiza los cambios
5. Commit: `git commit -m "feat: descripción de cambios"`
6. Push: `git push origin feature/tu-feature`
7. Abre un Pull Request

### Estilo de Código

- Sigue las convenciones de Dart (Effective Dart)
- Usa nombres descriptivos
- Comenta código complejo
- Ejecuta `flutter format .` antes de commit
- Ejecuta `flutter analyze` para verificar errores

### Estructura de Commits

- `feat:` - Nueva funcionalidad
- `fix:` - Corrección de bug
- `docs:` - Cambios de documentación
- `style:` - Cambios de formato
- `refactor:` - Refactorización de código
- `test:` - Pruebas
- `chore:` - Cambios de configuración

Ejemplo: `feat: add match timeline widget`

### Pull Request

- Proporciona una descripción clara
- Referencia issues relacionados
- Asegúrate que los tests pasen
- Documenta cambios significativos
